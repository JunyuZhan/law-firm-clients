🔍 路由直接访问测试

测试目的：验证生成的链接是否可以直接在浏览器中打开，无需手动输入到 Portal 页面。

测试步骤：

  1. 确保前端开发服务器正在运行（npm run dev）
  2. 确保后端服务正在运行
  3. 从后端获取一个真实的项目访问链接（格式：/portal/matter/{id}?token={token}）
  4. 点击下面的测试链接，或直接在浏览器地址栏输入完整URL
  5. 检查是否直接跳转到项目详情页，而不是 Portal 首页
  6. 打开浏览器控制台（F12），查看是否有路由匹配的日志

⚠️ 注意：请将下面的链接替换为从后端获取的真实链接

预期结果：
  • ✅ 直接显示项目详情页面（不是 Portal 首页）
  • ✅ 浏览器控制台显示：[Router] 匹配到项目详情路由
  • ✅ 浏览器控制台显示：[MatterDetail] 路由信息
  • ✅ 项目详情数据正确加载

如何获取真实测试链接：

  1. 通过管理后台创建一个项目，或
  2. 通过 API 推送项目数据，返回的 accessUrl 字段就是测试链接

如果测试失败：

  • 检查浏览器控制台的错误信息
  • 确认路由配置是否正确（/portal/matter/:id 应该在 /portal 之前）
  • 确认前端服务是否重新编译了最新代码
  • 检查后端 API 是否正常（/portal/api/matter/{id}?token={token}）
