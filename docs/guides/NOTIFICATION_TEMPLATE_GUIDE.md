## 什么是通知模板

通知模板是预定义的通知内容格式，用于统一管理短信、微信、邮件等通知的内容。通过模板，您可以：

- ✅ 统一管理通知内容格式
- ✅ 支持动态变量替换（如项目名称、访问链接等）
- ✅ 方便修改和维护通知内容
- ✅ 支持多个服务商（阿里云、腾讯云、微信等）

---

## 模板类型说明

系统支持三种类型的通知模板：

### 1. 短信模板 (SMS)

**适用场景**：发送短信通知给客户

**服务商**：
- `aliyun` - 阿里云短信
- `tencent` - 腾讯云短信

**必需字段**：
- `templateCode` - 模板代码（在服务商平台申请的模板ID）
- `signName` - 签名名称（在服务商平台申请的签名）

### 2. 微信模板 (WECHAT)

**适用场景**：发送微信公众号模板消息

**服务商**：
- `wechat` - 微信公众号

**必需字段**：
- `templateCode` - 模板ID（在微信公众平台申请的模板ID）

### 3. 邮件模板 (EMAIL)

**适用场景**：发送HTML格式邮件

**服务商**：无需指定（使用SMTP服务器）

**特点**：
- 支持HTML格式
- 支持变量替换

## 如何创建模板

### 方式一：通过管理后台创建

1. 登录管理后台
2. 进入 **系统管理** → **通知模板管理**
3. 点击 **新增模板** 按钮
4. 填写模板信息：
   - **模板名称**：给模板起个名字，如"项目更新通知"
   - **模板类型**：选择 SMS、WECHAT 或 EMAIL
   - **服务商**：选择 aliyun、tencent 或 wechat（邮件不需要）
   - **模板代码**：填写在服务商平台申请的模板ID
   - **签名名称**：短信模板需要填写签名名称
   - **模板内容**：填写模板内容（JSON格式，见下文）
   - **模板变量**：填写变量说明（可选，用于说明）
   - **是否启用**：选择是否启用该模板
   - **描述**：填写模板描述（可选）

5. 点击 **确定** 保存

### 方式二：通过API创建

```bash
POST /api/admin/notification-templates
Content-Type: application/json
Authorization: Bearer {JWT_TOKEN}

{
  "templateName": "项目更新通知",
  "templateType": "SMS",
  "templateCode": "SMS_123456789",
  "templateContent": "{\"code\":\"${matterName}\"}",
  "provider": "aliyun",
  "signName": "律师事务所",
  "enabled": true,
  "description": "项目更新时发送的短信通知"
}
```

## 模板格式详解

### 1. 短信模板格式

短信模板内容使用 **JSON格式**，格式取决于服务商：

#### 阿里云短信格式

```json
{
  "code": "${matterName}"
}
```

或者多个变量：

```json
{
  "code": "${matterName}",
  "url": "${accessUrl}"
}
```

**说明**：
- 变量名必须与阿里云平台申请的模板中的变量名一致
- 使用 `${变量名}` 格式表示变量

#### 腾讯云短信格式

```json
["${matterName}", "${accessUrl}"]
```

**说明**：
- 使用数组格式，按顺序对应模板中的变量
- 变量顺序必须与腾讯云平台申请的模板中的变量顺序一致

### 2. 微信模板格式

微信模板内容使用 **JSON格式**，必须符合微信模板消息格式：

```json
{
  "first": {
    "value": "项目信息已更新",
    "color": "#173177"
  },
  "keyword1": {
    "value": "${matterName}",
    "color": "#173177"
  },
  "keyword2": {
    "value": "${accessUrl}",
    "color": "#173177"
  },
  "remark": {
    "value": "点击查看详情",
    "color": "#173177"
  }
}
```

**说明**：
- `first` - 第一行内容（通常是标题）
- `keyword1`, `keyword2`, `keyword3` 等 - 关键词内容
- `remark` - 备注内容（最后一行）
- `value` - 内容值，支持变量替换
- `color` - 文字颜色（可选）

### 3. 邮件模板格式

邮件模板内容使用 **HTML格式**，支持变量替换：

```html
<!DOCTYPE html>
<html>
<head><meta charset='UTF-8'></head>
<body>
  <h2>项目信息已更新</h2>
  <p>尊敬的${clientName}，</p>
  <p>您的项目 <strong>${matterName}</strong> 信息已更新。</p>
  <p>请点击以下链接查看详情：</p>
  <p><a href="${accessUrl}">${accessUrl}</a></p>
</body>
</html>
```

**说明**：
- 支持完整的HTML格式
- 使用 `${变量名}` 格式表示变量
- 系统会自动替换变量

## 模板变量说明

系统支持以下预定义变量：

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `${matterName}` | 项目名称 | "合同审查项目" |
| `${accessUrl}` | 访问链接 | "https://example.com/portal?token=xxx" |
| `${clientName}` | 客户名称 | "张三" |

**变量替换规则**：
- 系统在发送通知时，会自动将模板中的变量替换为实际值
- 如果变量值为空，会替换为空字符串
- 变量名区分大小写

## 实际使用示例

### 示例1：创建阿里云短信模板

**步骤**：

1. 在阿里云短信平台申请模板，获得模板代码：`SMS_123456789`
2. 模板内容为：`您的项目${matterName}已更新，请查看：${accessUrl}`
3. 在系统中创建模板：

```json
{
  "templateName": "项目更新短信通知",
  "templateType": "SMS",
  "templateCode": "SMS_123456789",
  "templateContent": "{\"matterName\":\"${matterName}\",\"accessUrl\":\"${accessUrl}\"}",
  "provider": "aliyun",
  "signName": "律师事务所",
  "enabled": true,
  "description": "项目更新时发送的短信通知"
}
```

**说明**：
- `templateContent` 中的变量名必须与阿里云平台模板中的变量名一致
- 如果阿里云模板变量名为 `code`，则使用 `{"code":"${matterName}"}`

### 示例2：创建腾讯云短信模板

**步骤**：

1. 在腾讯云短信平台申请模板，获得模板ID：`123456`
2. 模板内容为：`您的项目{1}已更新，请查看：{2}`
3. 在系统中创建模板：

```json
{
  "templateName": "项目更新短信通知",
  "templateType": "SMS",
  "templateCode": "123456",
  "templateContent": "[\"${matterName}\",\"${accessUrl}\"]",
  "provider": "tencent",
  "signName": "律师事务所",
  "enabled": true,
  "description": "项目更新时发送的短信通知"
}
```

**说明**：
- `templateContent` 使用数组格式
- 数组元素的顺序必须与腾讯云模板中的变量顺序一致

### 示例3：创建微信模板

**步骤**：

1. 在微信公众平台申请模板，获得模板ID：`abc123def456`
2. 在系统中创建模板：

```json
{
  "templateName": "项目更新微信通知",
  "templateType": "WECHAT",
  "templateCode": "abc123def456",
  "templateContent": "{\"first\":{\"value\":\"项目信息已更新\",\"color\":\"#173177\"},\"keyword1\":{\"value\":\"${matterName}\",\"color\":\"#173177\"},\"keyword2\":{\"value\":\"${accessUrl}\",\"color\":\"#173177\"},\"remark\":{\"value\":\"点击查看详情\",\"color\":\"#173177\"}}",
  "provider": "wechat",
  "enabled": true,
  "description": "项目更新时发送的微信通知"
}
```

### 示例4：创建邮件模板

**步骤**：

1. 在系统中创建模板：

```json
{
  "templateName": "项目更新邮件通知",
  "templateType": "EMAIL",
  "templateContent": "<!DOCTYPE html><html><head><meta charset='UTF-8'></head><body><h2>项目信息已更新</h2><p>尊敬的${clientName}，</p><p>您的项目 <strong>${matterName}</strong> 信息已更新。</p><p>请点击以下链接查看详情：</p><p><a href=\"${accessUrl}\">${accessUrl}</a></p></body></html>",
  "enabled": true,
  "description": "项目更新时发送的邮件通知"
}
```

## 模板使用流程

### 1. 系统如何选择模板

当系统需要发送通知时，会按以下顺序选择模板：

1. **短信通知**：
   - 查询数据库中 `templateType=SMS` 且 `provider=当前配置的服务商` 且 `enabled=true` 的模板
   - 选择第一个匹配的模板
   - 如果找不到模板，使用默认格式

2. **微信通知**：
   - 查询数据库中 `templateType=WECHAT` 且 `provider=wechat` 且 `enabled=true` 的模板
   - 选择第一个匹配的模板
   - 如果找不到模板，使用默认格式

3. **邮件通知**：
   - 使用固定的HTML模板格式（当前版本）
   - 未来版本可能支持数据库模板

### 2. 变量替换过程

1. 系统读取模板内容
2. 解析模板中的变量（如 `${matterName}`）
3. 从项目数据中获取实际值
4. 替换模板中的变量
5. 发送通知

## 常见问题

### Q1: 如何知道模板代码（templateCode）？

**A**: 模板代码需要在对应的服务商平台申请：

- **阿里云短信**：登录阿里云控制台 → 短信服务 → 国内消息 → 模板管理 → 创建模板 → 获得模板代码
- **腾讯云短信**：登录腾讯云控制台 → 短信 → 国内短信 → 模板管理 → 创建模板 → 获得模板ID
- **微信公众号**：登录微信公众平台 → 功能 → 模板消息 → 我的模板 → 从模板库中选择 → 获得模板ID

### Q2: 短信模板的变量名必须与平台一致吗？

**A**: 是的！

- **阿里云**：`templateContent` 中的变量名必须与阿里云平台模板中的变量名完全一致
- **腾讯云**：`templateContent` 数组的顺序必须与腾讯云平台模板中的变量顺序一致

### Q3: 可以创建多个相同类型的模板吗？

**A**: 可以，但系统只会使用第一个启用的模板。建议：

- 为不同场景创建不同的模板
- 通过 `enabled` 字段控制使用哪个模板
- 或者修改代码逻辑，根据条件选择不同的模板

### Q4: 模板内容格式错误怎么办？

**A**: 

- **短信模板**：确保JSON格式正确，变量名与平台一致
- **微信模板**：确保符合微信模板消息格式
- **邮件模板**：确保HTML格式正确

如果格式错误，系统会：
- 记录错误日志
- 使用默认格式发送通知
- 不会影响通知发送功能

### Q5: 如何测试模板？

**A**: 

1. 创建模板后，确保 `enabled=true`
2. 在系统中触发一次通知（如接收项目数据）
3. 查看通知记录，确认是否使用了模板
4. 检查实际发送的内容是否符合预期

### Q6: 模板变量有哪些？

**A**: 当前支持的变量：

- `${matterName}` - 项目名称
- `${accessUrl}` - 访问链接
- `${clientName}` - 客户名称

未来可能会增加更多变量。

## 最佳实践

1. **模板命名规范**：
   - 使用清晰的名称，如"项目更新短信通知"
   - 包含模板类型和服务商信息

2. **模板内容规范**：
   - 保持内容简洁明了
   - 避免使用特殊字符
   - 测试变量替换是否正确

3. **模板管理规范**：
   - 创建模板后先禁用，测试无误后再启用
   - 定期检查模板是否过期
   - 保留模板历史记录

4. **多模板策略**：
   - 为不同场景创建不同模板
   - 使用描述字段说明模板用途
   - 通过启用/禁用控制使用哪个模板
