# 部署后冒烟测试流程

本文档用于客户服务系统首次部署、升级后复测或客户交付验收，目标是验证核心链路可用：

- 管理后台登录
- 客户门户访问
- 项目详情打开
- 文件链路可用
- 通知查询可用
- 函件验证可用

## 1. 测试目标

本轮冒烟测试覆盖以下主链路：

1. 基础服务可用
2. 管理员可登录后台
3. 客户门户首页可打开
4. 项目访问链接可正常进入详情页
5. 文件列表、预览或下载链路正常
6. 通知记录与通知配置页面可打开
7. 函件验证公开入口可访问

## 2. 测试环境

按实际环境填写，建议至少记录：

- 前端入口
- 后端入口
- 部署目录
- 当前部署版本
- 镜像标签
- Git 提交号

## 3. 基础健康检查

验证项：

- `GET /actuator/health` 返回 `UP`
- 前端首页可返回 `200`
- 后台登录页可返回 `200`

## 4. 后台链路

验证项：

- 管理员账号可登录
- 项目列表可打开
- 文件管理可打开
- 系统信息页可加载运行时信息与依赖状态
- 系统维护页可加载状态、备份和日志区块

## 5. 门户链路

验证项：

- `/portal` 可访问
- 粘贴项目访问链接后可进入详情页
- 项目详情中的文件和状态信息正常显示
- 帮助中心可访问

## 6. 函件验证链路

验证项：

- `/verify/letter` 可访问
- 输入有效函件编号和验证码后返回正确状态
- 无效或过期状态能被清晰展示

## 7. 推荐命令

### 7.1 快速 API 冒烟

```bash
./scripts/test/run-all-tests.sh quick
```

### 7.2 前端构建校验

```bash
cd frontend
npm run build
```

### 7.3 后端测试

```bash
./scripts/test.sh all
```

## 8. 通过标准

以下条件同时满足，视为本轮部署通过：

- 基础健康检查通过
- 后台登录与核心管理页正常
- 门户首页与项目访问链路正常
- 函件验证页正常
- 快速 API 冒烟无阻断性失败
